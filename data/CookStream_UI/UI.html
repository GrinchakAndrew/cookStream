CookStream UI

Screen 1

  • UI 1
    • Epic: Sign-up flow
      As: a User, when I am on the UI #1
      I want: to see the logo of the application
      And: The name of the application
      And: be able to sign up via FB account or with my email And: User can skip to 'Connect with Phone Number' option on the next screen, if user has no Facebook account

    • User-story: Sign-up | Sign-in via phone-number:
      When: I input my phone-number Then: I can validate the number via received sms
      And: And input my security code
      And: And input my again via 'Send Code again' clickable anchor

    • User-story: Sign-up | Sign-in via phone-number:
      When: I input my phone-number Then: I should pick my Country form the dropdown options
      And: See the aerial code of my country preset
    • User-story: Sign-up | Sign-in FB account:
      When: I on on UI # 1
      And: I want to sign up | sign-in via my FB account
      Then: I can do so via login to my FB account in the application

    • User-story: Sign-up | Sign-in via telephone number:
      When: user inputs his telephone number
      And: And user inputs his security code
      Then: he can register | sign-in to the app to create account | to log in

    • User-story: Logging in via phone number
      When: User logs in
      Then: He should receive sms with the pass code
      And: user can sign in to the application via the input of the received security code from the sms
    • User-story: Logging in via FB account
      When: User logs in
      Then: He should log in with his FB account
    • User-story:
      When: User has deleted the application from his device
      And: user has re-installed the app again
      Then: User will see the sign up screen to input the phone-number and receive the sms with the security code
      Or: Log in with his FB account
    • When : User logs in via Facebook account
      Then : User has mandatory input for his Facebook account
      And: User cannot skip this input
    • NB!
      - The keyboard should be implemented consistently to UI: the native keyboard should be blocked.
      - Countries should be selectable fro the dropdown list of items, made scrollable and should not overlap the UI keyboard.
