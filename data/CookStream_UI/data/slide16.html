    PUSH Notifications:
    • User-story: PUSH notifications -> questionnaire never started upon | left off in the middle
      When : User has never begun to pass the questionnaire
      And \ Or : User has not completed the questionnaire, but started passing it
      Then : User should receive an invite to start passing the questionnaire as PUSH notification
      And \ Or : User should receive an invite to continue passing the questionnaire
      And : the frequency of the PUSH notifications should be ONCE a week.
      NOT IN THIS SCOPE :
      Make the frequency variable for PUSH notifications configurable on the server-side to change periodicity of PUSH-notifications.

    • User-story: PUSH notifications -> new videos added
      When : no less then 10 videos have been added to the stream
      Then : PUSH notification is dispatched to the user
      And : the text is: "New Videos Added"
      And : click on the PUSH notification redirects the user to the stream, to the topmost video

    • User-story: PUSH notifications -> purchase of the next week
      When : user has NOT subscribed within 5 days
      And : the current week's subscription has expired
      Then : PUSH notification is dispatched to the user
      And : the text is: "Next Week Subscription Pending"
      And : click on the PUSH notification redirects the user to the online website for payment via Stripe

    • User-story: PUSH notifications -> new category added
      When : new category has been added
      Then : PUSH notification is dispatched to the user
      And : the text is: "Check our new category"
      And : click on the PUSH notification redirects the user to the added category view

    • User-story: PUSH notifications -> user did not log in to application for more than 3 days: invite to sign in
      When : user did not log in to application for more than 3 days
